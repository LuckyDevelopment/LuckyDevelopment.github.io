1. Camera Stuttering

This bug is really annoying, but when turning corners the camera lags and stutters a lot. Also when moving it seems to be slightly visible. It's actually pretty simple to fix it. The camera that I was using in my game was CinemachineVirtualCamera and a CinemachineBrain. Here is the hierarchy for my player object.

Player
 \ PlayerModel
 \ SoundHolder
 \ Orientation
 \ CinemachineCamera
 \ CameraHolder
   \ MainCinemachineCamera

You could just use a camera inside a camera holder if you want. With my testing though, I used Cinemachine Cameras since they come with extra features. Here is a code segment for some of my player controller code that has the stuttering bug, this script is located on the Player object from the hierarchy above.


// === Code segment from working player script but has camera stuttering/lag. ===

// PUBLICS
[Header("Objects")]
[SerializeField] private Tranform Orientation; 
[SerializeField] private Tranform CameraHolder; 
[SerializeField] private Tranform PlayerModel;

[Header("Settings")]
[SerializeField] private float MovementSpeed = 1f;
[SerializeField] private float CameraSensitivity = 300f;

// PRIVATES
private Rigidbody rb;
private Vector2 cameraRotation = Vector2.zero;

// Start is called once at start of game.
void Start()
{
    // Get the main rb component.
    rb = GetComponent();
}

// Handle the camera methods.
void LateUpdate()
{
    HandleCamera();
}

// Handle physics methods.
void FixedUpdate()
{
    HandleMovement();
}

// Handle camera, should be called in LateUpdate.
void HandleCamera()
{
    // Get the correct inputs.
    cameraRotation.x += Input.GetAxis("Mouse X") * CameraSensitivity;
    cameraRotation.y += Input.GetAxis("Mouse Y") * CameraSensitivity;
    
    // Limit looking upwards past 90 degrees
    cameraRotation.y = Mathf.Clamp(cameraRotation.y, -90, 90);
    
    // Apply the rotation
    CameraHolder.rotation = Quaternion.Euler(cameraRotation.x, cameraRotation.y, 0);
    Orientation.rotation = Quaternion.Euler(cameraRotation.x, cameraRotation.y, 0);
    PlayerModel.rotation = Quaternion.Euler(cameraRotation.x, cameraRotation.y, 0);
    transform.rotation = Quaternion.Euler(cameraRotation.x, cameraRotation.y, 0);
}

// Handle movement, should be called in FixedUpdate
void HandleMovement()
{
    // Get inputs
    float Horizontal = Input.GetAxisRaw("Horizontal") * MovementSpeed * Time.deltaTime;
    float Vertical = Input.GetAxisRaw("Vertical") * MovementSpeed * Time.deltaTime;
    
    // Apply inputs
    rb.AddForce(Orientation.forward * Vertical + Orientation.right * Horizontal, ForceMode.Force);
}

One line of code causes this problem, and it's actually very simple. It's actually the transform.rotation in the camera method. The thing is, we rotate the tranform of the object the script is on. The problem is the camera is under that transform, so when we rotate the camera holder that is under the player transform, they conflict with each other. We need to remove this transform.rotation line. Here is the final working code below.


// === Working code segment from working player script ===

// PUBLICS
[Header("Objects")]
[SerializeField] private Tranform Orientation; 
[SerializeField] private Tranform CameraHolder; 
[SerializeField] private Tranform PlayerModel;

[Header("Settings")]
[SerializeField] private float MovementSpeed = 1f;
[SerializeField] private float CameraSensitivity = 300f;

// PRIVATES
private Rigidbody rb;
private Vector2 cameraRotation = Vector2.zero;

// Start is called once at start of game.
void Start()
{
    // Get the main rb component.
    rb = GetComponent();
}

// Handle the camera methods.
void LateUpdate()
{
    HandleCamera();
}

// Handle physics methods.
void FixedUpdate()
{
    HandleMovement();
}

// Handle camera, should be called in LateUpdate.
void HandleCamera()
{
    // Get the correct inputs.
    cameraRotation.x += Input.GetAxis("Mouse X") * CameraSensitivity;
    cameraRotation.y += Input.GetAxis("Mouse Y") * CameraSensitivity;
    
    // Limit looking upwards past 90 degrees
    cameraRotation.y = Mathf.Clamp(cameraRotation.y, -90, 90);
    
    // Apply the rotation
    CameraHolder.rotation = Quaternion.Euler(cameraRotation.x, cameraRotation.y, 0);
    Orientation.rotation = Quaternion.Euler(cameraRotation.x, cameraRotation.y, 0);
    PlayerModel.rotation = Quaternion.Euler(cameraRotation.x, cameraRotation.y, 0);
    // REMOVED TRANSFORM.ROTATION LINE
}

// Handle movement, should be called in FixedUpdate
void HandleMovement()
{
    // Get inputs
    float Horizontal = Input.GetAxisRaw("Horizontal") * MovementSpeed * Time.deltaTime;
    float Vertical = Input.GetAxisRaw("Vertical") * MovementSpeed * Time.deltaTime;
    
    // Apply inputs
    rb.AddForce(Orientation.forward * Vertical + Orientation.right * Horizontal, ForceMode.Force);
}

This code works because before the transform of the player and the camera conflicted, now they do not. You will need a seperate model under the main transform that you have to rotate for the player model if you want to have one.
